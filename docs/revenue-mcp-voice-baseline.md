# 收益 MCP 语音触发基线

更新时间：2026-04-16

## 目标

这份文档只记录当前已经验证可用的语音侧收益链接法，避免后续继续偏回 `LLMConfig` 自由调用或重新放大误触发范围。

当前原则：

- 收益 / 复盘 / 调价策略 一律走 `Hotel` 后端路由后再调用 MCP
- 不直接把收益 MCP 挂到 `LLMConfig`
- 真执行类继续保持更高门槛；当前主开放的是：
  - `run_operating_summary`
  - `run_store_revenue_review`
  - `generate_current_pricing_strategy`

## 当前后端链路

```text
用户语音
-> ASR 文本
-> /api/rtc/sessions/{id}/utterances
-> RevenueMCPService.resolve_query()
-> 收益 MCP / 收益后端
-> 结果播报 + 飞书发送
```

关键实现文件：

- `/Users/sky/Project/Hotel/app/services/revenue_mcp.py`

## 重复触发保护

以下三类有副作用的工具，短时间内只允许触发一次：

- `run_operating_summary`
- `run_store_revenue_review`
- `generate_current_pricing_strategy`

短时幂等窗口：

- `12` 秒

## 飞书发送规则

当前默认：

- 收益分析 / 复盘 / 调价策略 默认发送飞书

只有句子里明确出现这些否定说法，才不发送：

- `别发群`
- `不要发群`
- `不用发群`
- `不发群`
- `别推送`
- `不要推送`
- `不用推送`
- `不推送`
- `别发飞书`
- `不要发飞书`
- `不用发飞书`
- `不发飞书`

## 当前基线结论

- 收益分析：已验证可稳定触发
- 当前调价策略：已验证可稳定触发
- 昨日复盘：必须优先识别回顾语义，禁止误落当前策略

后续调整原则：

- 不再把收益 MCP 改回 `LLMConfig` 直调
- 不随意放宽收益链提前接管词
- 不删除复盘优先规则
- 不删除副作用工具幂等保护
